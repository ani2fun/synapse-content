# Control Flow

So far your programs have run straight down the page, one line after another. Tier 1 adds the two powers that make a program a *program*: **deciding** (do this only if that is true) and **repeating** (do this for every item, or until something changes). With those, plus lists to hold many values and functions to name a piece of work, you can write genuinely useful code.

Six chapters, in order:

1. [**Booleans, Comparisons & Logic**](/synapse/programming-languages/python/control-flow/booleans-and-logic) — the True/False values that every decision rests on, and how to combine them.
2. [**Conditionals**](/synapse/programming-languages/python/control-flow/conditionals) — `if`/`elif`/`else`: running code only when a condition holds.
3. [**Loops**](/synapse/programming-languages/python/control-flow/loops) — `while` and `for`: repeating work without copy-paste.
4. [**Loop Control & Patterns**](/synapse/programming-languages/python/control-flow/loop-control-and-patterns) — `break`, `continue`, the loop `else`, and the accumulator patterns you'll reuse forever.
5. [**Lists, the Basics**](/synapse/programming-languages/python/control-flow/lists-the-basics) — storing many values in order, and changing them in place.
6. [**Functions, the Basics**](/synapse/programming-languages/python/control-flow/functions-the-basics) — naming a reusable piece of work, with inputs and a returned result.

This tier leans on **indentation** — Python uses it to mark which lines belong to an `if` or a loop — so watch it closely from Chapter 2 on. As before, every code block with a ▶ Run button is live: predict the output, then run it and check.
